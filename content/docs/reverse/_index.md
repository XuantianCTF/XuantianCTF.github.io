逆向工程是分析程序的内部结构和逻辑，理解程序如何工作的技术。在 CTF 中，逆向方向主要考察对二进制程序的分析和破解能力。

### 什么是逆向工程？

逆向工程（Reverse Engineering）是指将编译后的二进制程序还原为可读的代码和逻辑。就像拆解一台机器来理解它的运作原理一样，逆向工程师通过反汇编、调试等手段来分析程序的行为。

逆向工程需要扎实的汇编基础和耐心的分析能力。通过学习逆向，你将深入理解计算机底层运行机制，掌握程序调试和漏洞挖掘的核心技能。

### 为什么学习逆向工程？

1. **深入理解计算机**：逆向让你真正理解程序是如何运行的
2. **漏洞挖掘**：许多高级漏洞（如 0day）需要逆向分析能力
3. **恶意软件分析**：安全公司需要大量逆向工程师分析病毒木马
4. **知识产权保护**：帮助企业发现盗版和侵权行为

### 核心知识点

| 知识领域 | 内容 |
|----------|------|
| 汇编语言 | x86/x64 指令集、函数调用约定 |
| 调试技术 | 断点、单步、内存查看、寄存器操作 |
| 工具使用 | IDA Pro、Ghidra、x64dbg、GDB |
| 算法识别 | 加密算法、编码方式、校验算法 |
| 文件格式 | PE/ELF 结构、DWARF 调试信息 |

### 学习路线

1. 学习 x86/x64 汇编语言基础
2. 掌握 IDA Pro、Ghidra 等反汇编工具
3. 熟练使用调试器（x64dbg、GDB）
4. 了解常见加密算法和编码方式
